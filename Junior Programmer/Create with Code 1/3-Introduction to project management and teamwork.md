# 摘要

设计和开发界的大多数工作都需要团队协作，需要用项目管理和设计工具来确保项目成功发行。在这个教程中，我们将为你介绍项目规划、生产、项目管理和团队合作中的关键概念和最优技术



# 总览

项目管理和团队合作一直都被认为是设计和开发行业的“软技能”。尽管项目管理可以通过学习技术，有许多不同的项目管理方法学可以选择，想变成一个好的项目管理者，最主要的还是团队合作——包括交流和管理技能。

高效的项目管理和团队合作包含：

- 和团队成员、客户和股东的清晰交流
- 在截止日前发行
- 能够符合项目预期和预定目标的最终产品

在这个教程，你将学习一些管理项目和团队合作的最佳产业实务。



# 回顾制作阶段

个人、团队、公司和管理人员都需要有特定的项目管理方法，而大多的项目都可以被划分为几个阶段。

正如你之前所学的那样，产品周期可以被划分为前期制作，制作，后期制作和维护四个阶段。合理规划每个阶段对于按时保质完成一个项目至关重要。



# 项目规划总览

当需要完成一个项目的时候，你需要规划日程表，管理时间、交流和合作。即便是独立项目，你也需要在制作过程中的某些阶段和他人交流，尤其是在进行**[用户测试](https://learn.unity.com/tutorial/introduction-to-user-feedback)**的时候。

让我们来学习一些成功完成和交付项目的指导，不管你是作为个人还是团队中的一员。



## 手册和追踪

手册在项目管理中扮演着重要的职能。**设计手册**是唯一真理，你可以通过设计手册记录你想创建的体验中的特殊需求。设计手册同样可以帮助创作者和项目管理者追踪设计和开发进程。

在管理项目的时候，始终按下面的方法来使用设计手册，从而指导设计和开发过程：

- 明确目标、受众和项目目的
- 通过创建项目计划，明确必要的项目步骤
- 使用项目计划，创建项目日程，明确具体的用时和交付日期
- 始终追踪关键里程碑，从而按时交付
- 在团队工作中分配职责，为每个团队成员指定任务
- 确保你和所有的团队成员都能够完成自己的任务。



## 时间管理

有效的时间管理对按时完成和交付项目至关重要。在管理项目的时候：

- 估算设计和开发过程中每个阶段的用时
- 每天检查项目计划，确保每天的进展汇报到项目管理人和制作者
- 为不可预见的延误流出弹性计划；有必要的话，重新优化和更新任务和交付节点
- 明确哪个项目计划和管理工具更适合你或者你的团队；始终使用你最开始选择的工具。



## 交流

如果没有清晰的交流，合作将变得极其困难。在任何领域，交流都是与他人合作所需要的最重要的技能，对于有效的项目管理也同样重要。

当你为了项目和其他人交流的时候：

- 搞清楚你的进展，以及可能影响他人的问题，不管它们是团队伙伴还是外部合作者。
- 尊重你和他人的时间
- 当评价别人的工作时，切记建设性和感性，务必让你的反馈有效、具体、尊重。
- 积极听取和采纳反馈，对反馈的处理要真诚回复



## 尊重和专业

即使你要完成的项目是个人项目，你也需要尊重他人和在工作中秉持专业态度。

当你在项目中与他人合作的时候：

- 要守时
- 向合作者、同事和客户做出回应
- 听取他人的观点和建议
- 在团队工作中积极奉献



# 设计文档和项目规划

在前期制作阶段，创建设计文档来帮助明确和估计创作目标。这些文档包含：

- 游戏设计文档（Game Design Documents）
- 目标人群（Target user persons）
- 项目章程（Project charter）
- 技术说明（Technical specifications）



## 设计文档

设计文档包含项目的蓝图，蓝图包含：

- 宏观总览；例如，项目的整体构图
- 特定项目流水线的需求和标准
- 特定功能的设计细则



在宏观设计文档中，你需要明确：

- 项目的目标
- 预想用户
- 项目的关键功能
- 最终产品

设计文档可以包括示意图，流程图，框架，类似项目中的启发图示，描述目标观感的参考图像。

如果你还没有一份设计文档，你可以下载一份[模板](https://connect-prd-cdn.unity.com/20201117/99fd6c6b-0d44-42f5-9373-eb8a1887cacf/Project%20Design%20Doc%20TEMPLATE%20-%20[GDOC].pdf)



## 项目章程

如果已经创建好设计文档，并且项目被批准制作，就需要项目章程了。这是描述整个项目的正式文档，包括项目的目标，如何实现，以及 股东是谁。项目章程在前期制作和制作阶段都会用到。

你可以下载项目章程的一个[示例模板](https://connect-prd-cdn.unity.com/20201117/bb422f61-9b42-4521-9828-910aae22e39c/Example%20of%20Project%20Charter.pdf)

项目章程包含：

- 项目始因
- 项目目标和局限
- 主要利益相关者
- 风险
- 项目优势
- 整体预算



## 附加文档

根据你的项目，你可能还需要在前期制作阶段制作其他的支持文档来帮助你评估生产计划。你可以根据需要在前期制作和制作过程中迭代这些文档。这些文档包括：

- 技术文档：包括项目技术部分的具体架构和功能
- 会议记录：如果你在团队中工作，编写会议记录来记录团队工作、依赖、进展和阻碍是非常有效的
- 提案和宣传文档：如果你需要获得公司的认可，或者需要吸引投资者，就需要正式的提案和宣传文档。



# 管理项目和追踪进展

当你评估好项目后，就需要开始规划如何使用适当的日程和项目计划来管理项目了。尽管项目规划通常在前期制作中产生，为了追踪项目进展，项目规划通常会在制作阶段使用和迭代。

项目计划为设计文档和项目章程中确定的元素设定明确的任务和可交付的日期。制造商或者项目管理者会经常创建项目计划并在需要修改的时候更新计划。他们同样会创建项目追踪，项目追踪可以作为项目的一部分，也可以单独创建，用来比对当前的进展和预期的差别。

如果你不创建计划、或者不按计划进行，通常会导致偏离预估——你的计划里会不断添加新想法和新功能——最终导致项目失败，延期或者超出预算。

下面是一些成功追踪计划的指引：

- 明确必要的项目步骤
- 如果在团队中工作，明确并分配项目职责
- 创建有着具体交付内容和用时的时间线
- 明确项目管理中的常见问题，例如偏离预估或者过渡期待的设计计划，过于紧迫的交付时间
- 确定什么时候完成项目
- 为项目中的每个小部分都规划用时
- 为项目的每个部分创建合理的时间流
- 如果在团队中工作的话，为每项任务分配责任人。



## 为客户创建项目

如果你在为客户创建项目，下面是一些附加的建议，能够帮助你更有效地管理项目：

- 搞清楚项目职责——包括决定谁来做客户和团队间的联络员，项目不同元素的主要受益者。
- 理解客户的预期
- 做好计划来和客户有效交流——明确如何和客户交流，以及是否需要线上合作工具让客户检查项目。



# 练习：明确你用来支持项目的管理和追踪工具。

有很多项目管理工具可以帮助你明确和规划项目的每一步，它们提供多种功能来帮助个人和团队规划和追踪工作流，这样它们可以管理和按时交付。许多项目管理工具提供模板，你只需要按你的项目需求调整即可。

> 这里意思是你得学一下Git之类的工具，不然你会很吃亏



# 准备发布

在后期制作阶段，你及那个准备发布你的项目。你的项目计划需要说明质量担保测试（QA测试），内容编辑和检查，bug修复，以及追踪发布的交付性。

让我们多了解一下AQ测试。



## Quality assurance testing

在QA测试阶段，有两项非常重要的管理过程：QA测试脚本和bug回报过程



### 测试脚本

QA测试是一系列测试者需要遵循并报告测试结果的步骤。这些脚本经常在正式测试开始前被创建，用来提升产品的具体需求和用户场景。测试脚本也可能用来测试团队对玩家应有体验的假设。



### bug回报追踪

bug追踪和解决是发布项目的重要步骤。用来追踪bug报告的工具有很多——例如Github。

典型的bug报告包含：

- 标题/简短的描述：bug的描述性标题
- 频率：bug出现频繁吗，每次都有、还是大多时候有、偶尔或者很少出现？
- 复现步骤：清楚说明复现bug的步骤。这一项必须足够详细，这样开发者就可以自行复现bug
- 详细描述：更加详细描述bug，以及对游玩体验的影响
- 其他观察结果：包括对导致bug可能原因的猜测，以及其他相关细节（例如，错误不是每次都出现，但晚上出现比白天频繁）

查看https://www.ministryoftesting.com/articles/11b82aee了解更多有关bug回报的细节。



# 发行项目

在你测试过项目、修复bug并确认无误后，你已经准备好发布了。你的项目计划需要说明什么时间以及在哪发布你的项目——理想状况下，你赶上了交付日期，在你预想的时间发布了你的项目。

即使你没有赶上预期的日期，遵循清楚的项目计划并做出调整，你也可以得到更合理的发布日期。



# 运营活动和回顾

## 运营活动

制作周期的最后一个阶段是运营，包括发布一个产品后所有维持它运行的工作。尽管运营活动通常在发步日后才进行，在前期制作阶段就应当明确和规划这一阶段。这些活动包括——贩售、变现、更新以及后续服务。



## 项目回顾

项目推出后，需要做出计划进行回顾，这样你或者你的团队可以从设计和开发的过程中获得反馈。回顾可以帮助你明确你们做的好的和不好的部分，以及如何在下一个项目改进。

回顾的一个简单方法是，创建一个三行的表格，填写你认为符合：

- 要开始做的
- 不要再做的
- 要继续保持的

有很多工具和工作流可以帮助你回顾，你可以找一些适合你项目的。



# 总结

在这个教程中，你学习了基础的项目管理。这些知识在你工作的任何项目中都可以用到，不管是大项目还是小项目，实时开发还是任何类型的项目。

当你开始下一个计划的时候，使用这个教程学到的建议，从而保证你按时交付。试着为你的下一个项目设置一些文档和简单的项目计划和追踪。
